# Swift TCCC

iOS app for documenting Tactical Combat Casualty Care (TCCC)
encounters. Voice intake → on-device speech recognition →
deterministic state extraction (MARCH/PAWS, vitals, MOI) → DD Form
1380 sections + 9-line MEDEVAC + ZMIST handoff.

The app runs entirely on-device. It does not send data over the
network — there is no client for ATAK, MEDHUB, or any other
external system, and no telemetry, analytics, or cloud-based
transcription. Outputs are files (JSON, audio, transcript, CSV) that
the user shares manually via the iOS share sheet, and an offline QR
code generated locally for scanning by another device.

## Status

5/5 screens functional. 697 unit tests passing. Targets iPhone
17 Pro on iOS 26.2.

| Screen | Function | Limitations |
|---|---|---|
| 01 Live Capture | On-device ASR (Apple Speech, Parakeet alt-toggle), engine-extracted facts panel, 30 s pre-roll + 30 s tail | Parakeet model is a one-time download (~300 MB) on first use |
| 02 Vital Signs Log | DD 1380 Section C grid (4 timestamped columns × 7 rows), Interventions panel | Cells are read-only; tap-to-edit is on the backlog |
| 03 TCCC Card | Body diagram (anterior + posterior), MARCH (incl. Hypothermia + TBI sub-rows per 2026 §7/§8), PAWS, meds log, back-of-card §D-H scaffold | Some §D-H fields populated best-effort from extracted state |
| 04 MEDEVAC | Auto-populated 9-line, voice-readable transmit script | "Transmit" generates a script and logs an event; no over-the-air transmission |
| 05 Handoff | Encounter summary, timeline, SLM narrative + ZMIST, exports | DD-1380 PDF export not yet implemented; other exports use the iOS share sheet |

The clinical extraction layer is aligned to the 01 May 2026 TCCC
Guidelines and the DD Form 1380 (June 2014). Field inventories and
vocabulary live in `reference/rubric/extracted/` as JSON; the
extractor patterns and DD-1380-bound UI elements reference them
directly. See `CLAUDE.md` for the audit log.

## Architecture

```
TCCC_IOS/
├── TCCC_IOS.xcodeproj         # generated by xcodegen — never hand-edit
├── project.yml                 # source of truth for Xcode config
├── TCCC_IOS/                   # SwiftUI app target (thin)
│   ├── App/                    # AppState, lifecycle, exports
│   ├── Audio/                  # SpeechRecognizer, ParakeetTranscriptStream, TranscriptStream
│   ├── Chrome/                 # StatusStrip, PageHeader, Panel, FooterHints
│   ├── Components/             # Cross-screen views
│   ├── Design/                 # Theme tokens, typography, layout
│   ├── Intelligence/           # Foundation Models / TCCCLLMBackend wrappers
│   ├── Pager/                  # 5-screen swipe pager
│   └── Screens/                # 01-05 screen views
├── Packages/TCCCKit/           # local SPM — pure logic, testable in isolation
│   └── Sources/
│       ├── TCCCDomain/         # PatientState, MARCHState, Vitals, MGRS, enums
│       ├── TCCCExtractor/      # PatientStateEngine + 11 extractors + normalizer
│       ├── TCCCReports/        # 9-Line + ZMIST fallback generators
│       └── TCCCDesign/         # Shared design tokens (placeholder)
└── reference/
    └── rubric/extracted/       # DD 1380 + 2026 TCCC Guidelines as JSON
```

The clinical logic lives in `TCCCKit`. The app target is SwiftUI
views, view models, and iOS-specific plumbing (audio, Foundation
Models, file export). All extraction logic, regex patterns, and
report formatters are unit-testable without launching the simulator.

## Constraints

These shape the dependency choices in this repo:

1. **No networking at runtime.** No Wi-Fi, Bluetooth, Cellular, UWB,
   or NFC clients are wired. No analytics, no crash reporting, no
   telemetry. ASR uses `requiresOnDeviceRecognition = true`.
   Foundation Models is on-device by design. The Parakeet
   alt-backend performs one HTTPS GET on first activation
   (operator-gated, model cached locally) and never networks again.
2. Landscape-only, iPhone-only, locked in `project.yml`.
3. AES-256 at rest via `NSFileProtectionComplete` on the data
   directory.
4. Minimum hit target 44 pt for primary actions; long-press only
   for destructive actions, with a visual progress fill.
5. Default theme is Dark. Tactical (deep near-black with
   desaturated red accents) and Light are operator-selectable.

## Build

```bash
# regenerate the .xcodeproj from project.yml
xcodegen generate

# build for simulator
xcodebuild -project TCCC_IOS.xcodeproj -scheme TCCC_IOS \
  -destination 'generic/platform=iOS Simulator' \
  -configuration Debug build CODE_SIGNING_ALLOWED=NO

# run kit unit tests in isolation
cd Packages/TCCCKit && swift test
```

For device install, the development team ID in `project.yml` must
be set to a value the local Xcode toolchain can sign with:

```bash
xcodebuild -project TCCC_IOS.xcodeproj -scheme TCCC_IOS \
  -destination 'id=<device-uuid>' \
  -configuration Debug build CODE_SIGNING_ALLOWED=YES

xcrun devicectl device install app --device <device-uuid> \
  ~/Library/Developer/Xcode/DerivedData/TCCC_IOS-*/Build/Products/Debug-iphoneos/TCCC_IOS.app

xcrun devicectl device process launch --device <device-uuid> com.aarzamen.TCCCai
```

`xcrun devicectl list devices` lists connected device UUIDs.

## Demo path

1. Live Capture is the default screen. Tap **Load demo · GSW thigh**
   to seed the engine without speaking.
2. The EXTRACTED panel populates with engine-derived facts (MOI,
   hemorrhage location, vitals, classification, etc.).
3. Swipe through the five screens; each renders the same engine
   state.
4. On Handoff, **Narrative** and **ZMIST** invoke the on-device
   language model to produce prose summaries.
5. **JSON Encounter** and **Audio + Transcript** open the iOS
   share sheet with the corresponding files.
6. Hold the **Transmit** button for two seconds with **QR · OFFLINE**
   selected to render an offline QR code containing the patient
   JSON. The QR code is rendered locally; nothing is sent.
7. Footer **NEW** / **END** / **WIPE** drive the casualty
   lifecycle. Single tap shows a top-positioned confirmation banner.

## ASR backends

Settings → Audio · ASR offers two backends:

- **Apple Speech** (default). Uses `SFSpeechRecognizer` with
  `requiresOnDeviceRecognition = true`. No download required.
- **Parakeet (alt)**. Uses NVIDIA Parakeet TDT 0.6B v2 (English
  only) repackaged for CoreML by FluidInference's
  [FluidAudio](https://github.com/FluidInference/FluidAudio) Swift
  SDK. The CoreML bundle is downloaded on first activation
  (~300 MB, one HTTPS GET, gated behind a Settings tap) and cached
  in Application Support thereafter.

The same 30 s pre-roll + 30 s tail and the same in-Settings dynamic
mic gain (-20 to +20 dB) apply to both backends.
