# TCCC.ai iOS

Native SwiftUI port of the `TCCC_FEB_2026` Python prototype. Combat-medic
documentation app for field use: iPhone, landscape, chest-mounted, fully
offline. Voice intake → on-device ASR → deterministic state extraction
(MARCH/PAWS, vitals, MOI) → DD Form 1380 + 9-line MEDEVAC + ZMIST handoff.

## Hard product constraints (non-negotiable)

Check any new package/framework/import against these first:

1. **RF Ghost** — no Wi-Fi/Bluetooth/Cellular/UWB/NFC clients, no analytics,
   crash reporting, telemetry, or auto-update SDKs. ASR is on-device only
   (`requiresOnDeviceRecognition = true`); any LLM is on-device only. The
   *only* network calls in the app are operator-gated one-time model
   downloads for the alternate backends.
2. **Landscape-only, iPhone-only**, locked in `project.yml`. Do not regress.
3. **Data-at-rest protection** via `NSFileProtectionComplete` on the data
   directory (code-set on every write; not independently audited).
4. **Gloved-hand input** — min hit target 44 pt, primary actions 56–64 pt.
   Long-press only for destructive actions, with a visual progress fill.
5. **NVG-safe themes** — default Dark; Tactical (deep near-black + desaturated
   red) and Light operator-selectable. No auto-switch.
6. **Multi-casualty engine, single-casualty UI** (design §9). The engine keeps
   the patient dict; UI surfaces `PATIENT_1` only.

## Core invariants (these bind all future work)

- **Event-sourced state.** `PatientState` is a deterministic projection of an
  immutable `EncounterEvent` log: `patients == PatientStateEngine.project(log)`.
  `project(log)` is the restore authority and equivalence oracle;
  `LogEquivalenceTests` (`snapshot() == project(log)`) gate it. The fold
  **replays recorded deltas**, it does not re-run extractors.
- **Engine is the sole writer of `PatientState`. The LLM never mutates state**
  (structural invariant — `EngineInvariantTests` + an app-side structural test
  enforce it). Views may swap a transcript line's *text* for display; all state
  goes through engine methods.
- **Continuous encrypted persistence.** App-layer `EncounterStore` actor owns
  all I/O: per-casualty `Documents/encounters/<id>/events.jsonl` + a manifest,
  written via `ProtectedWrite` with `NSFileProtectionComplete`. Replay-on-launch
  (`TCCC_IOSApp .task → AppState.load() → engine.restore()`) recovers an
  in-progress encounter after a crash. New Casualty / End Care **archive**
  (never delete); WIPE (hold 3 s) **purges** the tree and re-arms a fresh
  casualty. TCCCKit stays in-memory-pure; all I/O is app-layer.
- **Provisional-replace ASR commit pipeline.** Loss-safe and identity-scoped:
  a committed line is *provisional* until a 2.0 s settle window closes; a
  refined recognizer echo replaces it in place and the engine supersedes that
  chunk via log-tail truncate-and-reapply (refined words win, originally-heard
  text retained for audit). No lexical de-dup heuristic. Spec/plan:
  `docs/superpowers/{specs,plans}/2026-06-28-asr-provisional-replace*`.
- **DD 1380 is the deliverable** (2026 §19), not a feature. Every shipping UI
  element populates a DD 1380 field or drives a MARCH/PAWS phase-status change.
- **Backend defaults are load-bearing.** Apple Speech ASR + Apple Foundation
  Models are the validated runtime defaults. Alt backends (Parakeet + Granite
  Speech ASR; LFM2.5 + Qwen 3 LLM) are functional but **experimental**,
  reachable behind Settings, and require operator-gated downloads/setup. Don't
  break or remove the defaults to chase the alt path.

## Conventions

- **No logic in the app target** — SwiftUI views, view models, and iOS
  plumbing (audio, inference, file export) only. Everything else is in TCCCKit.
- **Regex**: Swift Regex (iOS 16+) by default; `NSRegularExpression` for PCRE
  features (lookbehind, etc.).
- **Vitals are typed structs**, not dicts.
- **SF Symbols are temporary** for medical/MEDEVAC iconography (custom stroke
  icons per design §7 are backlog). Don't invest deep work in SF Symbol comps.
- **Test parity**: when porting a Python test, copy assertions verbatim, then
  add Swift-only tests on top.

## Build, test, run

```bash
# regenerate the project + build for simulator
cd /Users/ama/TCCC_IOS && xcodegen generate
xcodebuild -project TCCC_IOS.xcodeproj -scheme TCCC_IOS \
  -destination 'generic/platform=iOS Simulator' \
  -configuration Debug build CODE_SIGNING_ALLOWED=NO -skipMacroValidation

# package logic tests in isolation (fast, no simulator)
cd /Users/ama/TCCC_IOS/Packages/TCCCKit && swift test
```

`-skipMacroValidation` is **required** for CLI builds: `huggingface/AnyLanguageModel`
ships Swift macros that need explicit trust (the IDE's "Trust & Enable" once;
the flag from CLI). Without it every build errors on `AnyLanguageModelMacros`.
On a fresh machine also fetch the Metal toolchain once (MLX-Swift GPU kernels):
`xcodebuild -downloadComponent MetalToolchain` (~688 MB).

App-target tests run on a booted simulator (use a concrete device id; bare
`id=` / `generic/platform=iOS Simulator` fail for the test action):

```bash
xcodebuild test -project TCCC_IOS.xcodeproj -scheme TCCC_IOS \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
  -configuration Debug CODE_SIGNING_ALLOWED=NO -skipMacroValidation
```

Device build + install (free provisioning, team `XM6E4PUXTU`; 17 Pro coredevice
id `4FC3C1DC-B809-552A-B60F-B1723ADB45B8`):

```bash
xcodebuild -project TCCC_IOS.xcodeproj -scheme TCCC_IOS -destination 'generic/platform=iOS' \
  -configuration Debug -skipMacroValidation -allowProvisioningUpdates \
  -derivedDataPath /tmp/tccc-device-dd build
DEV=4FC3C1DC-B809-552A-B60F-B1723ADB45B8
xcrun devicectl device install app --device "$DEV" \
  /tmp/tccc-device-dd/Build/Products/Debug-iphoneos/TCCC_IOS.app
xcrun devicectl device process launch --device "$DEV" com.aarzamen.TCCCai
# pull on-disk encounters to inspect persistence:
xcrun devicectl device copy from --device "$DEV" --domain-type appDataContainer \
  --domain-identifier com.aarzamen.TCCCai --source Documents/encounters --destination /tmp/tccc-pull
```

The `devicectl` "No provider was found" warnings are harmless — install/launch
succeed regardless. The two "Load demo" seed buttons appear on the Live Capture
empty-state (transcript empty).
