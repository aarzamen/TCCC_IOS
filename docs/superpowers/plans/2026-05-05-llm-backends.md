# LFM2 + Qwen LLM Backend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement this plan task-by-task.

**Goal:** Replace the `.notImplemented` stubs in `LFM2LLMBackend` and `QwenLLMBackend` with real MLX-Swift inference via `huggingface/AnyLanguageModel`, so the operator can pick either model in Settings and have it actually generate radio scripts / ZMIST / narratives / cleaned transcripts.

**Architecture:** Add `AnyLanguageModel` SPM package with the `MLX` trait enabled. Implement one generic `MLXBackend` that wraps `AnyLanguageModel.MLXLanguageModel(modelId:)`. Refactor the two existing per-model stubs (`LFM2LLMBackend`, `QwenLLMBackend`) into thin adapters that forward to `MLXBackend` with their specific model ids and display names. Availability is computed by probing the HuggingFace Hub cache directory; when not cached, Settings shows a "Download" affordance that kicks off a pre-fetch.

**Tech Stack:** Swift 6.2 / iOS 17+ / SwiftUI / `huggingface/AnyLanguageModel >= 0.8.0` (with `MLX` trait) / `mlx-community/LFM2-1.2B-4bit` (660 MB) + `mlx-community/Qwen3-1.7B-4bit` (968 MB) downloaded on demand.

**Reference research:** `RESEARCH_LLM_SMALL.md` covers license, benchmarks, iPhone 17 Pro perf, integration alternatives, and the validator/diagnostics roadmap.

- [ ] **Step 2: Write `MLXBackend.swift`**

```swift
import Foundation
import AnyLanguageModel

/// Generic MLX-backed LLM backend. Wraps `AnyLanguageModel.MLXLanguageModel`,
/// which auto-downloads weights from Hugging Face Hub on first use and caches
/// them locally. The actor itself is stateless across `generate(...)` calls
/// (every call constructs a fresh `LanguageModelSession`) so context never
/// bleeds between casualties or between generation kinds — same invariant
/// `TCCCLanguageModel` already established for Apple Foundation Models.
///
/// Two concrete instantiations live in `LFM2LLMBackend` and `QwenLLMBackend`.
actor MLXBackend: TCCCLLMBackend {
    nonisolated let displayName: String
    private let modelId: String

    init(displayName: String, modelId: String) {
        self.displayName = displayName
        self.modelId = modelId
    }

    /// Reports whether the model snapshot is already cached locally. Cached =
    /// `.available`. Not cached = `.modelNotProvided` (the operator must tap
    /// "Download" in Settings to pre-fetch). The actual download happens
    /// inside `generate(...)` if it wasn't pre-fetched, so a `.modelNotProvided`
    /// status doesn't mean inference will fail — only that it'll block on a
    /// network call first.
    var availability: BackendAvailability {
        get async {
            HFHubCache.contains(modelId: modelId) ? .available : .modelNotProvided
        }
    }

    func generate(instructions: String, prompt: String) async throws -> String {
        let model = MLXLanguageModel(modelId: modelId)
        let session = LanguageModelSession(model: model, instructions: Instructions(instructions))
        do {
            let response = try await session.respond { Prompt(prompt) }
            return response.content
        } catch {
            throw BackendError.generationFailed(
                "\(displayName): \(error.localizedDescription)"
            )
        }
    }
}
```

If the actual `AnyLanguageModel.LanguageModelSession` initializer doesn't accept an `instructions:` parameter (the public API as of v0.8.0 may differ from the research-doc sketch — the README shows `LanguageModelSession(model:)` without instructions), then prepend the instructions to the prompt as a system-style block:

```swift
let response = try await session.respond {
    Prompt("\(instructions)\n\n\(prompt)")
}
```

— and add a doc comment noting the workaround. **Read AnyLanguageModel's `LanguageModelSession.swift` source before you write this** to confirm the real init signature; it's at `Sources/AnyLanguageModel/LanguageModelSession.swift` in the package's checkout (under DerivedData after the build).

- [ ] **Step 3: Add the HFHubCache helper**

In the same file (`MLXBackend.swift`), add:

```swift
/// Probes HuggingFace Hub's local snapshot cache. The cache is created and
/// populated by `mlx-swift-lm` (a transitive dep of AnyLanguageModel via the
/// MLX trait) using the standard `Hub.snapshot()` flow. Path layout:
///   ~/Library/Caches/huggingface/hub/models--<owner>--<repo>/snapshots/<rev>/
/// We don't need to know the revision — checking that the
/// `models--<owner>--<repo>` directory exists with at least one snapshot is
/// sufficient signal for "weights are on disk and inference will not block
/// on a download."
enum HFHubCache {
    static func contains(modelId: String) -> Bool {
        guard let url = directory(for: modelId) else { return false }
        let snapshotsDir = url.appendingPathComponent("snapshots", isDirectory: true)
        guard let entries = try? FileManager.default.contentsOfDirectory(atPath: snapshotsDir.path) else {
            return false
        }
        return !entries.isEmpty
    }

    static func directory(for modelId: String) -> URL? {
        let safe = modelId.replacingOccurrences(of: "/", with: "--")
        let key = "models--\(safe)"
        let caches = FileManager.default.urls(for: .cachesDirectory, in: .userDomainMask).first
        return caches?.appendingPathComponent("huggingface/hub/\(key)", isDirectory: true)
    }
}
```

The cache root is iOS-correct: `URLs(for: .cachesDirectory, in: .userDomainMask).first` resolves to `~/Library/Caches/<bundleId>/...` on iOS, which is where `Hub.swift` writes by default unless the host overrides `HF_HOME`. If first device test reveals weights are landing somewhere else (probable — Hub.swift may use a different default on iOS than macOS), update `directory(for:)` to match. Log the actual path on the device once and bake it.

- [ ] **Step 1: Add `prefetch()` to MLXBackend**

```swift
extension MLXBackend {
    /// Trigger weight download into the HF Hub cache without performing a real
    /// generation. We force this via a 1-token throwaway respond — the model
    /// loader has to materialize the weights before the first token, which is
    /// the only goal here.
    func prefetch() async throws {
        let model = MLXLanguageModel(modelId: modelId)
        let session = LanguageModelSession(model: model, instructions: Instructions("warmup"))
        // Not @discardableResult on purpose — we want the load completion
        // signal even though we throw the response away.
        var opts = GenerationOptions()
        opts.maximumResponseTokens = 1
        _ = try await session.respond(options: opts) { Prompt("hi") }
    }
}
```

Look up the actual `GenerationOptions` field name for max-response-tokens — could be `maxResponseTokens`, `maxNewTokens`, or `maximumResponseTokens` depending on the AnyLanguageModel version. Cross-reference with `Sources/AnyLanguageModel/GenerationOptions.swift`.

```swift
func downloadBackendWeights(_ backend: LLMBackend) async {
    guard downloadingBackend == nil else { return }
    downloadingBackend = backend
    defer { downloadingBackend = nil }

    let mlx: MLXBackend
    switch backend {
    case .lfm2:  mlx = MLXBackend(displayName: "Liquid LFM2.5 1.2B", modelId: "mlx-community/LFM2-1.2B-4bit")
    case .qwen3: mlx = MLXBackend(displayName: "Qwen 3 1.7B",        modelId: "mlx-community/Qwen3-1.7B-4bit")
    case .appleFoundation: return // nothing to download
    }
    do { try await mlx.prefetch() }
    catch { appendSystem("DOWNLOAD FAILED · \(backend.displayName) · \(error.localizedDescription)") }
}
```

(Watch for actor-isolation: this needs to be `@MainActor` since it touches `downloadingBackend`. Or use `MainActor.run { ... }` around the field updates.)
